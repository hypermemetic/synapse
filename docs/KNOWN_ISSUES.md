# Known Issues

## Synapse CLI Cannot Call plexus-derive Generated Methods

**Status**: Known Issue
**Severity**: High
**Affects**: Synapse 3.8.0
**Date Identified**: 2026-03-23

### Symptoms

When trying to call methods generated by the `#[activation]` procedural macro (plexus-derive), synapse CLI fails with:

```bash
$ synapse substrate ping pong --message "test"
Fetch error at ping: Protocol error: Subscription error:
RpcErrorObj {errCode = -32603, errMessage = "Internal error", errData = Nothing}
```

The `_self test` command also fails:
```bash
$ synapse _self test --raw '{"message":"hello"}' localhost 4444 substrate ping.pong
[FAIL] Test FAILED
Protocol error: Connection failed: Network.Socket.connect: <socket: 5>: does not exist (Connection refused)
```

### Root Cause

The methods **are accessible and working correctly** via direct JSON-RPC/WebSocket connections:

```bash
$ echo '{"jsonrpc":"2.0","id":1,"method":"ping_pong","params":{"message":"test"}}' | websocat ws://127.0.0.1:4444

# Response (works perfectly!):
{"jsonrpc":"2.0","id":1,"result":163261151425519}
{"jsonrpc":"2.0","method":"ping_pong","params":{"subscription":163261151425519,"result":{"type":"data",...}}}
{"jsonrpc":"2.0","method":"ping_pong","params":{"subscription":163261151425519,"result":{"type":"done",...}}}
```

**The issue is in synapse's transport/schema client layer**, not in the generated methods.

### What Works

✅ Direct WebSocket calls (websocat, wscat)
✅ Any standard JSON-RPC client
✅ Methods register correctly in substrate
✅ Subscriptions stream correctly
✅ Protocol compliance (single Done event)
✅ Schema generation for methods

### What Doesn't Work

❌ Synapse CLI calls to plexus-derive generated methods
❌ Synapse `_self test` on plexus-derive methods
❌ Schema fetching via synapse client

### Comparison: hub_methods vs #[activation]

**VERIFIED VIA TESTING**:

**hub_methods (old macro)**:
- Synapse CLI: ✅ **WORKS**
- Direct WebSocket: ✅ WORKS
- Example: `synapse substrate bash execute --command "echo hello"` → Success!

**#[activation] (new macro)**:
- Synapse CLI: ❌ **FAILS** (-32603 Internal error)
- Direct WebSocket: ✅ WORKS (websocat, Haskell test)
- Example: `synapse substrate ping pong --message "test"` → Fails!

Both generate working JSON-RPC methods at the protocol level. The difference is in the **schema metadata** or **method signatures** that synapse's client library depends on to build its IR (intermediate representation).

### Investigation Findings

**✅ PROVEN**: A minimal Haskell WebSocket test successfully calls plexus-derive methods!

Test: `test/WebSocketRaw.hs` - Does exactly what websocat does
- Uses same Haskell runtime as synapse
- Uses same `websockets` library as synapse
- Sends same JSON-RPC request
- Receives all three messages correctly (subscription ID, data, done)

**Result: TEST PASSED** ✅

This proves:
1. The macro-generated methods ARE accessible from Haskell
2. The JSON-RPC protocol is correct
3. The WebSocket transport works fine
4. **The issue is specifically in synapse's Transport/Client modules**

### Investigation Needed

The bug is isolated to synapse's client library code:

1. **Schema Fetching**: Synapse's `buildIR` fails when calling plexus-derive methods
2. **Subscription Setup**: The Transport layer errors before even sending the RPC call
3. **Client Code Path**: Something in `Synapse.Transport` or `Plexus.Client` breaks

The raw WebSocket test bypasses all of synapse's client abstraction and works perfectly.

Potential issues in synapse client code:
- Schema format validation rejects plexus-derive schemas
- Subscription parameter encoding in Transport module
- Connection pooling/reuse bug in Client module
- Type expectations mismatch in buildIR

### Workaround

Use direct WebSocket/JSON-RPC calls instead of synapse CLI:

```bash
# Instead of: synapse substrate ping pong --message "test"
# Use:
echo '{"jsonrpc":"2.0","id":1,"method":"ping_pong","params":{"message":"test"}}' | websocat ws://127.0.0.1:4444
```

Or use any JSON-RPC client library in your preferred language.

### Example Activation

```rust
// plexus-substrate/src/activations/ping/activation.rs
#[activation(
    namespace = "ping",
    version = "1.0.0",
    description = "Simple ping/pong test activation",
    plexus  // Enable Plexus JSON-RPC transport
)]
impl Ping {
    async fn pong(&self, message: String) -> impl Stream<Item = PingEvent> + Send + 'static {
        stream! {
            yield PingEvent::Pong { message };
        }
    }
}
```

**Generates**:
- `ping_pong` JSON-RPC method (works via WebSocket ✅)
- `ping_unsubscribe_pong` unsubscribe method
- Full jsonrpsee server trait implementation
- Schema metadata for method discovery
